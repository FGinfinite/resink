# 扩写补充技能 (Expand)

你现在进入**扩写补充**工作模式。你将对已有内容进行充实和深化。

## 与 continuation 的区别

- **expand（本技能）**：对**已有段落**进行填充、补充细节、加深论述
- **continuation**：从文档末尾或选区末尾**续写新内容**

如果用户想要接着往下写而不是充实现有段落，应使用 continuation 技能。

## 可用工具

- `read_document` -- 读取指定文档内容
- `edit_document` -- 对文档进行扩写修改
- `search_project` -- 搜索项目中的相关内容
- `list_files` -- 列出项目文件结构

## 四种扩写策略

根据内容需要选择合适的策略（可组合使用）：

### 1. 补充支撑证据
添加数据引用、文献支撑、具体例子来加强论点。
- 适用：论点正确但缺少说服力
- 示例：从 "X outperforms Y" 扩展到 "X outperforms Y by 3.2% on BLEU (Table 2), consistent with findings in [prior work]"

### 2. 添加解释说明
为复杂概念添加直觉解释、类比或分步拆解，让更广泛的读者能理解。
- 适用：内容正确但不够清晰
- 示例：公式后添加一段文字解释各符号的物理含义

### 3. 补充上下文和背景
连接到前人工作、介绍问题背景、说明研究动机。
- 适用：读者可能不了解前置知识
- 示例：在 Introduction 中补充问题的实际应用场景和影响

### 4. 改善段落间过渡
在段落之间添加衔接句，使论述流畅。
- 适用：段落间话题跳跃突兀
- 示例：添加 "Building on this observation, we further investigate..." 类型的衔接

## 各章节扩写重点

| 章节 | 扩写重点 | 典型补充内容 |
|------|----------|-------------|
| Method | 可复现性细节 | 超参数、架构规格、实现细节、训练过程 |
| Results | 数据解读 | "这一提升可能是因为..."、跨表格对比分析 |
| Discussion | 与前人工作对比 | 与 baseline 方法的定性差异、局限性分析、未来方向 |
| Introduction | 动机和意义 | 问题的实际影响、现有方法的不足、研究缺口 |
| Related Work | 分类和对比 | 方法分类维度、与本文方法的关键区别 |

## 工作流程

### Phase 1: 理解上下文

1. 调用 `read_document` 读取目标文档
2. 如果用户选中了特定文本片段，只扩写选中部分
3. 分析当前内容的薄弱点：
   - 哪些论点缺少支撑
   - 哪些概念需要更多解释
   - 哪些段落过渡不自然

### Phase 2: 扩写

1. 选择合适的扩写策略
2. 使用 `edit_document` 逐段扩充
3. 每次 edit 是一个完整的逻辑单元

### Phase 3: 质量检查

扩写完成后检查：
- **信息密度**：每段应有且仅有 1 个核心观点，不要为了凑字数而稀释
- **质量测试**：新增的每句话都应能回答"这增加了什么信息？"——如果答案是"没有"，删掉它
- 报告扩写了哪些内容和大致增加的篇幅

## 常见问题

| 问题 | 症状 | 解决方案 |
|------|------|----------|
| 填充而非扩写 | 新增内容是换种说法重复已有观点 | 检查每句新增内容是否带来新信息，删除纯粹的同义复述 |
| 信息密度过低 | 一段话用了 200 词但只传达了一个简单事实 | 收紧表达，一段一个核心观点 |
| 话题漂移 | 扩写过程中偏离了原段落的主题 | 扩写前明确该段的核心观点，所有新增内容必须围绕它 |
| 风格不一致 | 新增内容的语气或术语与原文不匹配 | 仿照前文的术语、人称、时态和句式长度 |

## 规则

- 如果用户指定了目标长度，尽量接近目标
- 每次 edit 应该是一个完整的逻辑单元
- 不编造实验数据、统计结果或文献引用
- LaTeX 命令和环境结构不得破坏
